1. Difference Between Local Storage And Session Storage

Local Storage

  • The data stored in it has no expiration date The storage limit is about 10MB
  • Its data is never transferred to the server Cons
  • Its data is plain text; hence it is not secure by design
  • The data type is limited to string; hence it needs to be serialized Data can only be read on the client-side, not on the server-side

Session Storage

  • It is preferred over cookies because it is much more intuitive than using cookies to store things.
  • Less back and forth to the server
  • A cookie is size limited to 4kb and using session storage you can have much more data saved per session
  • Faster page loads

2. Difference Between Global Scope And block Scope.

Global Scope

Global scope contains all of the things defined outside of all code blocks. A code block simply consists of grouped statements inside curly braces ({ }). if statement, loops, function are examples of structure that create a code block. A global variable has global scope. A global variable is accessible from anywhere in the code.

Blog Scope

block scope is similar to a function scope, but is limited to a block instead of a function.As of ES3, a catch clause in try / catch statements has a block scope, which means that it has its own scope. It's important to note that the try clause does not have a block scope, only the catch clause does.As of ES6, let and const variables are attached implicitly to the current block scope instead of the function scope. This means that these variables are limited to the block they were declared in, whether it's an if block, a for block, or a function.
